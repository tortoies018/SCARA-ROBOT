# SCARA 并联机器人控制器

**STM32C8T6 固件 + PyQt6 上位机**，控制两路 DM542 步进电机驱动 + SG90 舵机，实现并联 SCARA（五连杆）机器人的运动控制。

## 硬件架构

```
PC (PyQt6) ──USB/UART──→ STM32F103C8T6 ──TIM1_CH1(PA8)──→ DM542 #1 ──→ 步进电机1 (主动臂左)
                                         ├─TIM3_CH1(PA6)──→ DM542 #2 ──→ 步进电机2 (主动臂右)
                                         ├─TIM4_CH1(PB6)──→ SG90 舵机 (抬笔/下笔)
                                         ├─PA0 (AIN1) ──→ 光电开关1 (左限位)
                                         └─PA1 (BIN1) ──→ 光电开关2 (右限位)
```

### 机械参数
| 参数 | 值 |
|------|-----|
| 主动臂长 L1 | 110 mm |
| 从动臂长 L2 | 220 mm |
| 电机间距 D | 160 mm |
| 步进电机步数/圈 | 200 |
| DM542 细分 | 8 (1600 脉冲/圈) |
| 关节每圈脉冲 | 1600 (直连无减速) |

## 串口通信协议

**波特率**: 115200, 8N1  
**帧格式**: 文本行, `\r\n` 或 `\n` 结尾  
**编码**: ASCII

### 指令一览

| 指令 | 格式 | 说明 | 响应 |
|------|------|------|------|
| **位置模式** | | |
| `A` | `A θ₁ θ₂ spd` | **绝对移动** — 双电机转到指定角度 | `OK MOVE` → `RDY` |
| `M` | `M dθ₁ dθ₂ spd` | **相对移动** — 双电机从当前位置转动指定角度 | `OK MOVE` → `RDY` |
| **连续脉冲模式** | | |
| `O` | `O spd₁ spd₂` | **连续脉冲** — 符号=方向, 数值=速度(°/s), 独立控制两电机 | `OK PULSE` |
| `C` | `C` | **停止脉冲** — 停止所有电机 | `OK STOP` |
| **设置** | | |
| `V` | `V spd₁ spd₂` | **设置默认速度** (°/s)，分别设置两电机，后续指令 `speed=0` 时使用 | `OK SPEED` |
| `E` | `E 1/0` | **电机使能/禁能** | `OK ENABLED` / `OK DISABLED` |
| `SP` | `SP s1 s2` | **设置当前位置** (步数) | `OK` |
| **控制** | | |
| `P0` | `P0` | **抬笔** (舵机 0°) | `OK` |
| `P1` | `P1` | **下笔** (舵机 180°) | `OK` |
| `H` | `H` | **回零** — 双段式光电寻零 | `OK HOME` → `RDY HOME DONE` |
| `Q` | `Q` | **查询状态** | `POS s1 s2 RDY/BSY` |
| `!` | `!` | **紧急停止** | `OK STOP` |
| `ER` | — | 错误响应 | `ER BUSY` / `ER STOPPED` / `ER UNKNOWN` |

### 速度单位

所有速度参数均以 **°/s**（度/秒）为单位。固件内部自动换算为步进频率（Hz）：

```
frequency_hz = speed_dps × 1600 ÷ 360
```

范围: 5 ~ 720 °/s。

### 角度换算

```
step_count = angle_deg × 1600 ÷ 360
```

### 指令示例

```
V 90 90                      设置电机1=90°/s, 电机2=90°/s
E 1                          使能电机
A 90 90 90                   双电机转到 90° (竖直朝上), 速度 90°/s
M 10 -10 90                  电机1 +10°, 电机2 -10°
O 90 -90                     电机1正转 90°/s, 电机2反转 90°/s
C                            停止脉冲
Q                            查询: POS 400 400 RDY
```

### 响应格式

| 响应 | 含义 | 触发条件 |
|------|------|----------|
| `OK MOVE\r\n` | 运动指令已接受 | `M`/`A` 指令解析成功 |
| `RDY\r\n` | 单步运动完成 | 双电机步数均归零 |
| `OK PULSE\r\n` | 连续脉冲已启动 | `O` 指令解析成功 |
| `OK STOP\r\n` | 已停止 | `C`/`!` 指令 |
| `POS s1 s2 RDY\r\n` | 空闲状态位置报告 | `Q` 指令响应 |
| `POS s1 s2 BSY\r\n` | 运动中位置报告 | `Q` 指令响应 |
| `ER BUSY\r\n` | 拒绝指令 | 机器人正在运动/回零 |
| `ER UNKNOWN\r\n` | 未知指令 | 无法识别的命令 |

### DMA 串口

固件使用 **DMA + IDLE 中断** 实现串口收发：

- **接收**: `HAL_UARTEx_ReceiveToIdle_DMA` — 自动检测串口空闲，每帧数据触发一次回调
- **发送**: `HAL_UART_Transmit_DMA` — 非阻塞发送，静态缓冲区防冲突
- **缓冲**: 128 字节环形 FIFO，ISR 写入，主循环消费
- **命令处理**: 主循环 `SCARA_ProcessSerial()` 从 FIFO 读取字节，按 `\r`/`\n` 切分为命令行，逐个处理

## 运动控制

### DDA 速度协调

`SCARA_MoveRelative` 实现双电机 DDA（Digital Differential Analyzer）速度协调：

1. 计算两轴步数 `d1`, `d2`
2. 取最大步数 `N = max(|d1|, |d2|)`
3. 按比例分配速度: `speed1 = speed × |d1|/N`, `speed2 = speed × |d2|/N`
4. 两电机同时启停，实现关节空间直线插值

### 正运动学

给定两主动臂角度 `(θ₁, θ₂)`，求末端执行器坐标 `(X, Y)`：

1. 计算两主动臂末端位置 `P₁`, `P₂`（圆心）
2. 以 `L₂` 为半径求两圆交点
3. 取上方交点（机构向上翻转构型）

### 逆运动学

给定末端坐标 `(X, Y)`，求两主动臂角度 `(θ₁, θ₂)`：

1. 分别计算两电机到目标点的距离 `r₁`, `r₂`
2. 利用余弦定理计算夹角 `φ₁`, `φ₂`
3. 生成两组候选解 `α ± φ`
4. 通过 FK 闭环验证 (< 1mm) 选出正确解

### 功能模块

| 模块 | 功能 |
|------|------|
| **串口连接** | 端口选择、波特率、连接/断开 |
| **电机控制** | 使能/禁能、连续脉冲（正转/反转/停止） |
| **正运动学** | 输入 θ₁/θ₂ → 计算并显示末端坐标、发送 `A` 指令 |
| **逆运动学** | 输入 X/Y → 计算 θ₁/θ₂、同步到正运动学输入 |
| **轨迹绘制** | 直线插值 → IK 逆解 → 逐点 `A` 指令执行 + 进度条 |
| **画布** | 机械臂 2D 渲染、鼠标拖动平移、滚轮缩放 |
| **控制台** | 日志显示、清空、原始指令输入 |

### 轨迹执行流程

```
设置起点/终点/步长/速度 → 预览(笛卡尔插值)
  → IK 逆解为角度序列 → 角度连续化处理
  → 逐个发送 A 指令 → 等待 RDY → 下一发
  → 进度条实时更新
```
